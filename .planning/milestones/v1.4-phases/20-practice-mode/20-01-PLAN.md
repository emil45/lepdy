Create a NEW file `hooks/usePracticeSession.ts` — a dedicated practice hook that is completely separate from `usePuzzleSession.ts` (per research recommendation — avoids modifying tested challenge session hook, eliminates session storage corruption risk per Pitfall 3).

The hook signature:
```typescript
export interface UsePracticeSessionReturn {
  currentPuzzle: SessionPuzzle | null;
  consecutiveCorrect: number;
  onAnswer: (correct: boolean) => void;
  startPractice: (pieceId: ChessPieceId) => void;
  currentTiersByPiece: Record<string, PiecePuzzleProgress>;
}

export function usePracticeSession(): UsePracticeSessionReturn
```

Implementation details:
1. Import `SessionPuzzle` from `hooks/usePuzzleSession` (reuse the type, not the hook).
2. Import `movementPuzzles, capturePuzzles` from `data/chessPuzzles`.
3. Import `selectNextPuzzle, defaultGeneratorState` from `utils/puzzleGenerator`.
4. Import `usePuzzleProgress` from `hooks/usePuzzleProgress`.
5. Import `chessPieces, ChessPieceId` from `data/chessPieces`.

Internal state: `queue: SessionPuzzle[]`, `headIndex: number`, `consecutiveCorrect: number`, `genState: GeneratorState`, `activePieceId: ChessPieceId | null`.

`startPractice(pieceId)` function:
- Sets `activePieceId = pieceId`
- Calls internal `buildPracticeBatch(pieceId, getSessionTier)` to generate 5 puzzles
- Resets `headIndex = 0`, `consecutiveCorrect = 0`, `genState = defaultGeneratorState()`

`buildPracticeBatch(pieceId, getSessionTier)` internal function:
- Gets `tier = getSessionTier(pieceId)`
- Filters `movementPuzzles` to those with `pieceId === pieceId` AND filters `capturePuzzles` to those with `correctPieceId === pieceId`
- Generates 3 movement + 2 capture puzzles (or as many as available) using `selectNextPuzzle` on the filtered pools
- Returns `SessionPuzzle[]`
- IMPORTANT: If a filtered pool is empty for a piece (e.g., some pieces may have fewer capture puzzles), use only movement puzzles for that piece. Use `Math.min(targetCount, filteredPool.length)` to avoid infinite loops.

`onAnswer(correct)` callback:
- NO session-size cap (no `headIndex >= SESSION_SIZE` guard — this is practice mode, per locked decision)
- Extract `pieceId` from current puzzle (same pattern as `usePuzzleSession.ts` line 176-177)
- Call `recordCorrect(pieceId)` or `recordWrong(pieceId)` on the progress hook
- Update `consecutiveCorrect` (correct ? prev + 1 : 0)
- On correct: advance `headIndex`. When `headIndex >= queue.length`, generate a fresh batch via `buildPracticeBatch` and reset `headIndex = 0`
- On wrong: do NOT advance (child retries same puzzle, same as challenge mode)

NO sessionStorage persistence — practice sessions are ephemeral. No writes to `SESSION_STORAGE_KEY` or any other storage key.

`currentPuzzle` derivation: `queue[headIndex] ?? null` (no `SESSION_SIZE` cap check).
