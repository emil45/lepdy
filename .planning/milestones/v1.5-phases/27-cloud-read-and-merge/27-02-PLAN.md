    Create `hooks/useMergeOnSignIn.ts` with the following structure:

    1. **Constants:**
       - `SESSION_KEY = 'lepdy_merge_done'` — sessionStorage key
       - `STORAGE_KEYS` object mapping category names to localStorage keys:
         ```
         letters: 'lepdy_letters_progress'
         numbers: 'lepdy_numbers_progress'
         animals: 'lepdy_animals_progress'
         games: 'lepdy_games_progress'
         words: 'lepdy_word_collection'
         streak: 'lepdy_streak_data'
         ```
       - `RTDB_PATHS` array: `['progress/letters', 'progress/numbers', 'progress/animals', 'progress/games', 'progress/words', 'streak']`

    2. **fetchCloudData(uid: string)** async function:
       - Dynamic import `@/lib/firebase` for `getFirebaseDatabase`
       - Dynamic import `firebase/database` for `ref`, `get`
       - `Promise.all` to fetch all 6 RTDB paths under `users/{uid}/`
       - Return object mapping path to data (or null if snapshot doesn't exist)
       - Wrap in try/catch, log errors, return null on failure

    3. **readLocalStorage()** function:
       - Read all 6 localStorage keys, JSON.parse each
       - Return object mapping category name to parsed data (or null)
       - Wrap each parse in try/catch

    4. **writeLocalStorage(merged)** function:
       - JSON.stringify and write each merged result to its localStorage key
       - Wrap in try/catch

    5. **runMerge(uid: string)** async function:
       - Call fetchCloudData(uid)
       - If cloud fetch failed (returned null), exit early (no merge possible)
       - Call readLocalStorage()
       - Call each merge function from `@/lib/mergeProgress`:
         - `mergeCategoryProgress(local.letters, cloud['progress/letters'])` for letters, numbers, animals
         - `mergeGamesProgress(local.games, cloud['progress/games'])`
         - `mergeWordCollection(local.words, cloud['progress/words'])`
         - `mergeStreak(local.streak, cloud['streak'])`
       - Call writeLocalStorage with merged results
       - Call `window.location.reload()` to force context providers to re-read merged data

    6. **useMergeOnSignIn(user: User | null, loading: boolean): void** hook:
       - `prevUidRef = useRef<string | null | undefined>(undefined)` — undefined means "not yet resolved"
       - `useEffect` with deps `[user, loading]`:
         - If `loading` is true, return (wait for auth to resolve)
         - Extract `prevUid` from ref, set `currentUid = user?.uid ?? null`
         - Detect sign-in transition: `currentUid !== null && (prevUid === undefined || prevUid === null) && prevUid !== currentUid`
         - Update ref: `prevUidRef.current = currentUid`
         - If not a sign-in transition, return
         - Check sessionStorage: if `sessionStorage.getItem(SESSION_KEY) === currentUid`, return (already merged)
         - Call `runMerge(currentUid).then(() => sessionStorage.setItem(SESSION_KEY, currentUid))`

    IMPORTANT: Use the `'use client'` directive at top of file.
    IMPORTANT: Type `user` parameter as `User | null` importing `type { User } from 'firebase/auth'`.
    IMPORTANT: Do NOT write to RTDB in this hook. The existing `useProgressSync` in each context provider will pick up the merged localStorage data after reload and write it to RTDB via the debounced path.
