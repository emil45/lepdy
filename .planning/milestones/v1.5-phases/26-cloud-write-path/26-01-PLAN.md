2. Create `hooks/useProgressSync.ts` with:
   - `'use client'` directive at top
   - Export a function `useProgressSync(uid: string | null, path: string, data: unknown): void`
   - The `path` parameter is the RTDB sub-path under `users/{uid}/` (e.g., `'progress/letters'` or `'streak'`)
   - When `uid` is null, the hook must be a complete no-op — return immediately before any effect logic runs
   - Use `useRef` for debounce timer (30_000ms = 30 seconds)
   - Use `useRef` to hold latest data reference (`dataRef.current = data`)
   - Use `JSON.stringify(data)` as the serialized key for the `useEffect` dependency array to avoid infinite re-renders from unstable object references (see Research pitfall 1). Store the serialized string in a variable `const serialized = JSON.stringify(data)` and use `serialized` in the dep array.
   - Inside the effect: if `!uid` return early. Clear any existing timer. Set a new 30s timeout that:
     a. Dynamically imports `getFirebaseDatabase` from `@/lib/firebase`
     b. Dynamically imports `{ ref, set }` from `firebase/database`
     c. Calls `const db = await getFirebaseDatabase()`
     d. Calls `await set(ref(db, \`users/\${uid}/\${path}\`), dataRef.current)`
     e. Wraps everything in try/catch, logging errors with `console.error(\`[sync:\${path}] Write failed:\`, error)`
   - Cleanup function: clear the timeout on unmount or dependency change
   - IMPORTANT: Do NOT import firebase/database at the top level — dynamic import only inside the setTimeout callback to prevent SSR errors (matches existing pattern in lib/firebase.ts)
  </action>
