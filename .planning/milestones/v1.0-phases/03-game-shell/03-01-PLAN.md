  <action>
    Create `hooks/useChessProgress.ts` following the useCategoryProgress load-on-mount/save-on-change pattern exactly.

    **Exported interface:**
    ```typescript
    export interface ChessProgressData {
      completedLevels: number[];
      currentLevel: number;
    }

    export interface UseChessProgressReturn {
      completedLevels: number[];
      completeLevel: (levelNum: number) => void;
      isLevelUnlocked: (levelNum: number) => boolean;
      isLevelCompleted: (levelNum: number) => boolean;
    }
    ```

    **Storage key:** `'lepdy_chess_progress'` as a `const STORAGE_KEY`.

    **Implementation details:**
    - `useState<ChessProgressData>` initialized to `{ completedLevels: [], currentLevel: 1 }`
    - `useState<boolean>` for `isInitialized`, starts `false`
    - Load useEffect: `typeof window === 'undefined'` guard, try-catch around `localStorage.getItem(STORAGE_KEY)`, parse JSON, validate `Array.isArray(parsed.completedLevels)`, set data, set `isInitialized = true`
    - Save useEffect: depends on `[data, isInitialized]`, skips if `!isInitialized`, try-catch around `localStorage.setItem(STORAGE_KEY, JSON.stringify(data))`
    - `completeLevel` wrapped in `useCallback`: adds levelNum to completedLevels if not already present, sets currentLevel to `Math.max(prev.currentLevel, levelNum + 1)`
    - `isLevelUnlocked` wrapped in `useCallback`: returns true if levelNum===1, else checks `data.completedLevels.includes(levelNum - 1)`. Depends on `[data.completedLevels]`
    - `isLevelCompleted` wrapped in `useCallback`: returns `data.completedLevels.includes(levelNum)`. Depends on `[data.completedLevels]`
    - Error logging prefix: `'[chess]'` — matching `console.error('[chess] Failed to load progress:', e)` and `console.error('[chess] Failed to save progress:', e)`

    Note: This is simpler than useCategoryProgress — no migration logic, no storageError state, no config parameter. Direct hook with hardcoded storage key.
