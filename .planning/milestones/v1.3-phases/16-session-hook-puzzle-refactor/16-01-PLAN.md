Create `hooks/usePuzzleSession.ts` with `'use client'` directive.

**Exported types:**
```typescript
export type SessionPuzzle =
  | { type: 'movement'; puzzle: MovementPuzzle }
  | { type: 'capture'; puzzle: CapturePuzzle };

export interface UsePuzzleSessionReturn {
  currentPuzzle: SessionPuzzle | null;
  sessionIndex: number;       // 0-based, 0-9
  consecutiveCorrect: number; // session-scoped streak
  isSessionComplete: boolean; // true when sessionIndex reaches 10
  onAnswer: (correct: boolean) => void;
  startNewSession: () => void;
}
```

**Hook signature:** `export function usePuzzleSession(): UsePuzzleSessionReturn`

**Session queue generation (buildSessionQueue):**
- Generate 10 puzzles: 5 movement + 5 capture, interleaved (mov, cap, mov, cap...)
- For movement slots: rotate through `chessPieces` sorted by `order` (king, rook, bishop, queen, knight — 5 pieces for 5 slots). Use `getSessionTier(pieceId)` for each.
- For capture slots: pick a random piece from `chessPieces` for each slot. Use `getSessionTier(pieceId)` for each.
- Use `selectNextPuzzle(movementPuzzles, tier, genState)` and `selectNextPuzzle(capturePuzzles, tier, genState)` respectively, threading `GeneratorState` through each call.
- Return the queue as `SessionPuzzle[]`.

**sessionStorage persistence:**
- Key: `'lepdy_chess_session'`
- Stored shape: `{ queue: Array<{ type: 'movement' | 'capture'; id: string }>, headIndex: number, consecutiveCorrect: number }`
- Store only puzzle IDs, not full objects. On hydration, resolve IDs via `movementPuzzles.find(p => p.id === id)` and `capturePuzzles.find(p => p.id === id)`.
- If any ID lookup fails during hydration, discard persisted session and generate fresh.
- SSR guard: `typeof window !== 'undefined'` before accessing sessionStorage.
- Save to sessionStorage on every state change (headIndex, consecutiveCorrect) via useEffect.

**onAnswer(correct) implementation:**
- Extract `pieceId` from current puzzle: for movement use `puzzle.pieceId`, for capture use `puzzle.correctPieceId`.
- Call `recordCorrect(pieceId)` or `recordWrong(pieceId)` from `usePuzzleProgress`.
- Update streak: `setConsecutiveCorrect(prev => correct ? prev + 1 : 0)` (functional form to avoid stale closure per Pitfall 1).
- Advance `headIndex` to next puzzle: `setHeadIndex(prev => prev + 1)`.
- When `headIndex` reaches 10 (all answered), the session is complete. Do NOT call `completeLevel` here — that will be wired by Plan 02 in ChessGameContent.

**startNewSession:**
- Clear sessionStorage key.
- Generate fresh 10-puzzle queue.
- Reset headIndex to 0, consecutiveCorrect to 0.

**State initialization flow (useEffect on mount):**
1. Check sessionStorage for existing session.
2. If valid: restore queue (resolve IDs), headIndex, consecutiveCorrect.
3. If missing/invalid: call buildSessionQueue, set headIndex=0, consecutiveCorrect=0.
4. Set `isInitialized=true` flag.

**Internal hooks consumed:**
- `usePuzzleProgress()` for `getSessionTier`, `recordCorrect`, `recordWrong`
- `defaultGeneratorState()` and `selectNextPuzzle()` from `utils/puzzleGenerator`
- `chessPieces` from `data/chessPieces`
- `movementPuzzles`, `capturePuzzles` from `data/chessPuzzles`
