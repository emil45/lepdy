**Key difference: `displayFen` needs to extract piece-placement from full FEN.**
The Chessboard component takes piece-placement-only FEN for `position`. Extract it: `puzzle.fen.split(' ')[0]` for the initial displayFen, and use `moveFenPiece` on the piece-placement portion only for animation.

**Derived values:**
- `matingPieceConfig = chessPieces.find(p => p.id === puzzle.matingPieceId)!`

**Two-tap interaction (handleSquareClick):**
```
if (isAdvancing) return;

if (!selectedPieceSquare):
  if square === puzzle.matingPieceSquare:
    setSelectedPieceSquare(square)  // highlight the piece
  else:
    return  // ignore taps on other squares

if (selectedPieceSquare):
  if square === puzzle.targetSquare:
    // CORRECT — animate, celebrate
    setIsAdvancing(true)
    const piecePlacement = puzzle.fen.split(' ')[0]
    setDisplayFen(moveFenPiece(piecePlacement, puzzle.matingPieceSquare, puzzle.targetSquare))
    setFlashSquare(puzzle.matingPieceSquare)
    setFlashType('correct')
    setShowCorrectConfetti(true)
    playRandomCelebration()
    setTimeout(() => { resetFeedbackState(); onAnswer(true); }, 1500)
  else if square === puzzle.matingPieceSquare:
    // Tapped same piece again — deselect
    setSelectedPieceSquare(null)
  else:
    // WRONG target
    onAnswer(false)
    setFlashSquare(square)
    setFlashType('wrong')
    setShowTryAgain(true)
    setSelectedPieceSquare(null)  // reset selection
    setTimeout(() => { setFlashSquare(null); setFlashType(null); }, 600)
    setTimeout(() => { setShowTryAgain(false); }, 1200)
```

**squareStyles (useMemo):**
- When `selectedPieceSquare` is set: yellow glow on mating piece square (`boxShadow: 'inset 0 0 0 4px rgba(255, 215, 0, 0.7)'`)
- Flash correct: green background on mating piece square
- Flash wrong: orange background on wrong square

**Instruction text: Use `t('ui.tapToCheckmate')` — the translation key from Plan 01.**

**Checkmate confirmation overlay:** When `isAdvancing` is true, show:
- Large Typography with `t('ui.chessmate')` ("!שח מט") — bold, green color, centered, above the piece name
- Hebrew piece name + audio button (same pattern as CapturePuzzle's piece name reveal)

**Layout: Same as CapturePuzzle** — exit button row, beige board card with instruction + board, feedback text below, confetti overlay.

**Important: Do NOT use chess.js in the component for move validation.** The data is pre-validated. The component trusts puzzle.matingPieceSquare and puzzle.targetSquare. (Research anti-pattern: "Validation belongs in the data script, not the React component.")

**Board position prop:** Use `puzzle.fen.split(' ')[0]` (piece-placement only) as the initial position since the Chessboard component expects piece-placement FEN, not full FEN.
