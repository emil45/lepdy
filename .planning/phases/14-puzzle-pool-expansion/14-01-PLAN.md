Create `scripts/validate-puzzles.ts` — a Node.js script run via `npx tsx scripts/validate-puzzles.ts`.

The script must:

1. **Import puzzle data** from `../data/chessPuzzles` (movementPuzzles, capturePuzzles) and Chess from `chess.js`.

2. **Implement `getValidTargets(pieceOnlyFen: string, pieceSquare: string, isKingPuzzle: boolean): string[]`** using the dummy-king pattern from RESEARCH.md:
   - Parse the 8-rank FEN to find occupied squares
   - If NOT a king puzzle, insert white king `K` at `a1` (or `h1` if a1 is occupied)
   - Always insert black king `k` at `h8` (or `a8` if h8 is occupied)
   - Build full FEN: `{ranks} w - - 0 1`
   - Call `new Chess(fullFen).moves({ verbose: true, square: pieceSquare })`
   - Return deduplicated target squares: `[...new Set(moves.map(m => m.to))]`

3. **Validate each movement puzzle:**
   - Compute valid targets via `getValidTargets(puzzle.fen, puzzle.pieceSquare, puzzle.pieceId === 'king')`
   - Sort both computed and stored arrays, compare — report mismatch if different
   - Check `difficulty` is 1, 2, or 3
   - Check `id` is a non-empty string

4. **Validate each capture puzzle:**
   - Build FEN with all pieces (including target, correct piece, distractors) + dummy kings
   - Verify correct piece on `correctPieceSquare` CAN reach `targetSquare` via `moves({verbose:true, square: correctPieceSquare})`
   - Verify every distractor square CANNOT reach `targetSquare`
   - Check `difficulty` is 1, 2, or 3

5. **Check pool counts** (report warnings, not errors, if below thresholds — existing pool is small):
   - Total movement puzzles (target: 60)
   - Movement puzzles per piece (target: 10 each for king, queen, rook, bishop, knight, pawn)
   - Movement puzzles per difficulty tier (target: 10 each for tiers 1, 2, 3)
   - Total capture puzzles (target: 30)
   - Capture puzzles per piece as correctPieceId (target: 5 each)
   - Capture puzzles per difficulty tier (target: 10 each)

6. **Output format:**
   - Print each puzzle ID with PASS or FAIL + reason
   - Print summary: `{N} puzzles checked, {M} errors, {W} warnings`
   - Exit code 0 if no errors, exit code 1 if any errors
   - Warnings are for count shortfalls only (not blocking — existing pool will be small until Plan 02)

7. **Check for duplicate IDs** across movement puzzles and across capture puzzles.
